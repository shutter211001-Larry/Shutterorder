**定位**：次世代開源自託管餐飲營運、點餐與 ERP 系統 (一體化解決方案)

## 一、 系統完整功能掃描解析 (Full Feature Scan)

經過深度掃描專案目錄、後端路由 (`packages/api-server/src/routes`)、Prisma 資料庫結構與前端架構，本專案已具備完整企業級餐飲 ERP 與 POS 系統的龐大規模。以下為掃描萃取的完整功能矩陣：

### 1. 多渠道點餐與訂單管理 (Omnichannel Ordering)
* **全方位點餐模式 (`order.routes`)**：支援內用 (Table)、外送 (Delivery)、外帶自取 (Pickup) 與預約訂餐。
* **企業級團購系統 (`group-order.routes`)**：支援多人揪團點單、發起人管理與個別結算。
* **智慧購物車與結帳**：自動計算稅額、外送費、時段限制檢查與客製化備註。
* **廚房顯示系統 (KDS)**：基於 Socket.IO 的即時訂單看板 (Kanban)，一鍵切換餐點狀態（接單、製作中、出餐）。
* **藍牙出單整合**：前端直接整合 Web Bluetooth API (`bluetoothPrinter.ts`)，無縫連接硬體熱感應印表機。

### 2. 多門店與靈活菜單管理 (Menu & Location Ops)
* **多門店營運 (`location.routes`)**：單一後台管理多品牌/多分店，獨立設定營業時間、外送範圍與配送費。
* **高階菜單引擎 (`menu.routes`)**：支援無限層級分類、複雜客製化選項 (加料、少冰半糖)、過敏原標示。
* **時段與庫存控制**：支援特定餐期 (Mealtime) 專屬菜單，並具備即時庫存扣抵與售完自動隱藏機制。
* **桌位管理 (`reservation.routes`)**：視覺化桌位圖、線上預約訂位、時段衝突檢查與入座管理。

### 3. CRM 會員經營與行銷增長 (Marketing & Loyalty)
* **全通路會員制 (`customer.routes`, `loyalty.routes`)**：跨店會員資料整合、消費點數累積、會員等級機制。
* **多樣化促銷模組 (`coupon.routes`)**：百分比折扣、固定金額扣抵、免運費，支援最低消費限制與使用次數上限。
* **LINE 深度生態系整合 (`line.routes`)**：LINE OA 綁定、LINE 內點餐與 LINE Pay 支付，打造私域流量。
* **行銷自動化 (`automation.routes`)**：基於觸發條件的自動化訊息推播 (如：生日賀禮、挽回流失客)。
* **評價反饋系統 (`review.routes`)**：訂單完成後的 1~5 星評分與留言，後台提供審核與回覆機制。

### 4. 人事與排班 ERP (HR & Staff Management)
* **角色與權限控制 (`staff.routes`, `job-role.routes`)**：細粒度的權限配置 (Super Admin, Manager, Staff)，信箱邀請與單次安全令牌機制。
* **智慧考勤與排班 (`attendance.routes`, `roster.routes`)**：員工視覺化排班表、線上打卡系統、遲到早退異常標記與「考勤校正 (Correction)」審批流程。
* **請假與薪資輔助 (`leave.routes`)**：線上請假簽核流程、特休/病假額度計算。

### 5. 營收、財務與金流整合 (Finance & Payments)
* **多元支付整合 (`payment.routes`)**：Stripe 信用卡整合、現金結帳，並保留第三方支付擴展彈性。
* **電子發票與稅務 (`invoice.routes`)**：自動開立統一發票 (含載具、統編)，符合當地稅法合規性。
* **商業智慧儀表板 (`dashboard.routes`)**：即時營收趨勢圖、時段單量熱點、熱銷排行榜與分店業績對比。

### 6. 系統底層與企業級基礎設施 (Infrastructure)
* **國際化多語系 (`i18n.routes`)**：內建 13 種語言 (英、法、德、日、繁中等)，支援跨國餐飲品牌。
* **API 與開發者中心 (`developer.routes`, `integration.routes`)**：提供 Swagger API 文件，支援 Webhook 事件推送，方便串接外部 ERP 或外送平台 (UberEats/Foodpanda)。
* **安全性與法規 (`consent.routes`, `legal.routes`)**：隱私權政策同意記錄、系統操作稽核日誌 (Audit Log)。
* **多端點支援**：提供 Admin 總部後台、Storefront 顧客前台、ERP 前台，並具備 Mobile App 部署能力。

## 二、 產品市場定位與痛點解決 (Market Positioning)

### 目標客群 (Target Audience)
1. **中大型連鎖餐飲品牌**：需要跨店管理、統一數據與會員，且對資料隱私有強烈要求。
2. **注重品牌形象的獨立餐廳/咖啡廳**：需要美觀的線上點餐介面，取代無品牌的第三方平台。
3. **欲建立私域流量的商家**：希望擺脫外送平台 30% 以上高抽成，透過 LINE 或自有平台直接服務老顧客。

### 解決的市場痛點
* 🚫 **抽成過高**：使用 Shutter 自建點餐系統，實現 **0% 訂單抽成**，利潤完全回歸店家。
* 🚫 **系統破碎化**：將 POS、線上點餐、HR 排班、會員 CRM 四大系統統一，不再需要付費購買多套軟體並忍受資料不通的問題。
* 🚫 **會員資料被綁架**：自託管 (Self-hosted) 架構確保所有顧客數據與消費紀錄完全歸屬商家。

## 三、 階段性行銷推廣策略 (Marketing Strategy)

### Phase 1: 開源社群引爆與開發者口碑 (Launch & Developer Relations)
**目標**：在 GitHub 取得高星標，吸引開發者與技術型餐飲創業者的目光。
* **Product Hunt 發布**：主打「The Ultimate Open-Source Restaurant ERP & Ordering System」，製作高質感的展示影片，強調其可替代昂貴商業軟體 (如 Toast, Olo)。
* **技術內容行銷**：在 Dev.to, Medium, V2EX 等平台發表技術架構解析文章（如：「我們如何用 React + Node.js 打造零延遲的廚房 KDS 系統」）。
* **開源貢獻活動**：標記 `good first issue`，吸引全球社群協助翻譯 (已支援 13 國) 與串接當地支付閘道，形成全球生態系。

### Phase 2: 商業概念驗證與種子店家 (B2B Seed Acquisition)
**目標**：獲取首批真實餐飲店家使用，累積成功案例 (Case Studies)。
* **提供 Managed 託管服務 (SaaS 模式)**：雖然是開源軟體，但多數老闆不懂架設。推出「免安裝、月費制」的雲端代管服務 (Shutter Cloud)。
* **「拯救利潤」免費健檢活動**：針對當地餐廳投放 Facebook/IG B2B 廣告，提供「外送平台抽成試算表」，並免費為前 50 名商家建立專屬線上點餐網站。
* **功能亮點短影音 (Reels/TikTok)**：
  - 展示「顧客掃碼點餐 → 廚房 KDS 瞬間跳出訂單 → 藍牙印表機秒出單」的絲滑體驗。
  - 展示「排班異常手機一鍵校正」如何拯救店長的時間。

### Phase 3: 全渠道增長與生態系擴張 (Scale & Ecosystem)
**目標**：成為餐飲 SaaS 的主流選擇之一。
* **推薦分潤計畫 (Affiliate Program)**：邀請 POS 系統整合商、硬體設備商、餐飲顧問成為代理商，提供高額分潤。
* **App Store 上架**：利用專案內的 `mobile` 模組，協助旗艦級客戶打包專屬的品牌 iOS/Android App。
* **舉辦線上餐飲科技 Webniar**：主題為「後疫情時代的餐飲私域流量經營：如何用 LINE 與自動化行銷提升 30% 復購率」。
